A Tweets-controlled Python Script

Jul 26th, 2014 | Comments

Let us watch and react to the lattest tweets with Python, the dirty way.

Python modules like tweepy, python-twitter, twitter, or twython all depend on the Twitter API. As a consequence they require you to open a Twitter account, register at dev.twitter.com, create an app, and at each connection dance with the OAuth.

As an application, the following script watches my (useless) Twitter page, and each time I tweet something like cmd: my_command it executes my_command in a terminal:

I can now tweet-control, from my smartphone, any computer that is running this script. If I tweet cmd: firefox the computer will open firefox, if I tweet cmd: echo "Hello" it will print Hello in the terminal, etc.

Introducing Twittcher

If you want more, I wrote Twittcher, a small Python module which doesn’t depend on the Twitter API, to make bots that watch search results or user pages and reacts to the new tweets.

For instance this script checks the search results for chocolate milk every 20 seconds, and sends all the new tweets (with date, username, and link) to my mail box.

Just run that script all day on your computer (or rather on your Raspberry Pi) and you will be updated every time someone drinks chocolate milk and feels the urge to tweet about it (which is a LOT of times !).
